FHIR Conversion Mapping Rules

This tool converts HL7 v2.x messages to FHIR R4 resources. All major EHR vendors are supported (Epic, Cerner, Meditech, Allscripts, athenahealth, eClinicalWorks, NextGen).

Epic / EHR Identifier Type Mappings (CX-5 Type Code)

HL7 CX field format: value^check^authority^facility^type_code. The type code (5th component) determines the FHIR system URI.

CX Type CodeEHR MeaningFHIR system URI
EPIEpic Internal Patient IDurn:oid:1.2.840.114350.1.13
MRN / MRMedical Record Numberhttp://hospital.example.org/mrn
NPINational Provider Identifierhttp://hl7.org/fhir/sid/us-npi
SS / SSNSocial Security Numberhttp://hl7.org/fhir/sid/us-ssn
ANAccount Numberhttp://hospital.example.org/account
VNVisit Numberhttp://hospital.example.org/visit
ENEnterprise IDhttp://hospital.example.org/enterprise
PIPatient Internal ID (generic)http://hospital.example.org/patid
(authority only)No type code, authority in CX-3{system_prefix}:{authority}

Z-Segment Extensions (Epic / EHR Proprietary)

All Z-segment fields are preserved as FHIR extensions on the Patient resource so no proprietary data is lost.

ZEPEpic encounter propertiesurn:epic:z-segment:zep-{n}
ZALEpic allergy supplementurn:epic:z-segment:zal-{n}
ZDSDocument supplement (Cerner)urn:epic:z-segment:zds-{n}
ZCMCustom medication infourn:epic:z-segment:zcm-{n}
ZFX / ZIN / ZPI / ZSF / ZVS / ZPAVendor-specific fieldsurn:epic:z-segment:{name}-{n}
Any other Z*Auto-detected from messageurn:epic:z-segment:{name}-{n}
