# HL7 & EHR → FHIR Converter - Deployment Guide

## 🚀 Quick Start (Docker Hub)
```bash
# Pull and run from Docker Hub
docker run -d \
  --name hl7-fhir-converter-premium \
  -p 8000:8000 \
  joshvainva/hl7-fhir-converter:latest

# Access: http://localhost:8000
```

## 📦 Manual Transfer Method

### On Source Machine:
```bash
# Export the image
docker save hl7-fhir-converter:latest -o hl7-fhir-converter_premium.tar

# Transfer hl7-fhir-converter_premium.tar to target machine
```

### On Target Machine:
```bash
# Load the image
docker load -i hl7-fhir-converter_premium.tar

# Run the container
docker run -d \
  --name hl7-fhir-converter \
  -p 8000:8000 \
  --restart unless-stopped \
  hl7-fhir-converter:latest
```

## 🐳 Docker Compose Method

### Transfer these files to target machine:
- `docker-compose.yml`
- `.env` (if using environment variables)

```bash
# Run with docker-compose
docker compose up -d

# Check status
docker compose ps

# View logs
docker compose logs

# Stop
docker compose down
```

## 🎯 Supported Data Types
The converter now supports **both standard health messages and raw electronic health data**:

- **Standard HL7 v2.x**: ADT, ORU, ORM, SIU, MDM, DFT, VXU, MFN, ACK.
- **EHR Raw Data**: AI-powered conversion of clinical notes and summaries (Epic, Cerner, etc.).
- **Output Types**: Full FHIR R4 Bundle in JSON/XML, Styled Excel, CSV, and PDF Reports.

## ✨ Premium Configuration
- **Branding**: The application image automatically bundles the **Innova Golden Logo** and **Premium Dark UI**.
- **Security**: For production environments, ensure `ALLOWED_ORIGINS` is configured in your `.env` file to restrict access.
- **Auditing**: All exports generated by the deployed instance are automatically timestamped for audit tracking.
